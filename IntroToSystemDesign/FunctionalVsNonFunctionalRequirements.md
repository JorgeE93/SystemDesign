# Functional vs Non-Functional Requirements

In system design we need to define what the requirements for our system. We also need to define how or what these requirements need to function. 

## Functional Requirements 
Functional Requirements are what the functionality of the system is. This means listing out and describing what the system must do.

Example: 
- Authentication service must validate user credentials and provide session token with defined access level

## Non-Functional Requirements
Non-Functional Requirements are what the system needs in order to provide functionality.

Example:
- Scalability: The system has to be capable of growing with its user base
